## ⚙️ 进阶配置（可选）

### 🎯 调整推送模式

编辑 `config/config.yaml`：

```yaml
report:
  mode: "incremental"  # 改为增量模式，只推送新增新闻
  max_news_per_keyword: 5  # 每个词组最多5条
```

### 📊 控制显示数量

编辑 `config/frequency_words.txt`，在词组末尾添加：

```txt
物价
房租
油价
@3  # 这个词组最多显示3条新闻
```

### 🕐 设置推送时间

```yaml
notification:
  push_window:
    enabled: true
    time_range:
      start: "09:00"  # 早上9点开始
      end: "21:00"    # 晚上9点结束
```

## ❓ 常见问题

### Q1: 收到的新闻太多怎么办？
**A**: 在 `config.yaml` 中设置 `max_news_per_keyword: 3`

### Q2: 收到的新闻太少怎么办？
**A**: 
1. 放宽关键词限制（减少过滤词）
2. 增加更多平台
3. 调整为 `mode: "current"` 模式

### Q3: 某些词组没有新闻？
**A**: 正常情况，说明当天这个话题没有热点，可以：
- 等待其他时段
- 调整关键词范围
- 使用历史新闻库

### Q4: 如何避免收到重复新闻？
**A**: 将推送模式改为 `mode: "incremental"`

### Q5: 如何查看历史新闻？
**A**: 打开 `output/日期/html/` 文件夹，用浏览器打开 HTML 文件

## 🔧 故障排查

### 问题1: GitHub Actions 运行失败
- 检查是否配置了 Secrets
- 查看 Actions 日志中的错误信息

### 问题2: 没有收到推送
- 确认通知渠道配置正确
- 检查 `enable_notification: true`
- 确认关键词匹配到了新闻

### 问题3: 配置修改不生效
- 确认文件保存成功
- GitHub Actions 需要重新运行
- Docker 需要重启容器
